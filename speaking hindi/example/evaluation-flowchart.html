📊 Evaluation Flowchart

Tamil Reading Assessment - Complete Evaluation Pipeline

0
User Submits Answer
Website Frontend
  • User types answer in Tamil
  • Clicks "Evaluate" button
  • Answer sent to backend API
Input: {questionNumber, userAnswer}
⬇
1
Normalization
tamil_normalizer.normalize()
  • Unicode normalization (NFC)
  • Remove punctuation
  • Apply normalization map (variants → standard)
  • Remove extra spaces
Output: Normalized text (cleaned and standardized)
⬇
2
Semantic Similarity
tamil_semantic_engine.find_best_match()
  • Load multilingual embedding model
  • Encode user answer → embedding vector
  • Encode expected answers → embedding vectors
  • Compute cosine similarity for each pair
  • Find best match (highest similarity)
Output: semantic_score (0-1), best_match
Weight: 40% of final score
⬇
3
Concept & Logic Validation
tamil_concept_validator.validate()
3A: Concept Validation
  • Check required subject keywords
  • Check required action keywords
  • Check required object keywords
  • Check problem/time context (if needed)
3B: Logic Validation
  • Check for negation words
  • Validate if negation is allowed
  • Special check for Q5: detect wrong positive form
Output: concept_score (0-1), logic_score (0 or 1)
Combined Weight: 30% of final score
Formula: concept_score × 0.7 + logic_score × 0.3
⬇
4
Core Alignment
tamil_core_alignment.compute_best_alignment()
  • Extract concepts from user answer (subject, action, object, etc.)
  • Extract concepts from expected answers
  • Compare concept sets
  • Compute alignment score (matched concepts / total)
Output: alignment_score (0-1)
Weight: 20% of final score
⬇
5
Contradiction Detection
tamil_contradiction_detector.check_best_contradiction()
  • Check contradiction word pairs
  • Compare user answer with expected answers
  • If contradiction found → score = 0.0
  • If no contradiction → score = 1.0
Output: contradiction_score (0 or 1)
Role: Multiplier (if 0, entire score becomes 0)
⬇
6
Final Scoring
tamil_final_scorer.evaluate()
Weighted Sum:
0.40 × semantic_score +
0.30 × logic_score +
0.20 × alignment_score
Final Score:
weighted_sum × contradiction_score
Decision Threshold: 70% (0.70)

IF final_score >= 0.70:
  → marks = max_marks (FULL MARKS)
  → passed = True

ELSE:
  → marks = 0 (ZERO MARKS)
  → passed = False
Output: final_score, marks, passed
⬇
7
Return Results to Frontend
JSON Response
  • All 5 individual scores (as percentages)
  • Final score and marks
  • Pass/Fail status
  • Best match from expected answers
Output: Complete evaluation results
⬇
8
Display Results on Website
Frontend Display
  • Show Pass/Fail status (Correct/Incorrect)
  • Show marks (X/MAX)
  • Show all 5 scores with progress bars
  • Show best match

Score Weight Distribution

Semantic Similarity
40%
Logic Validation
30%
Core Alignment
20%
Contradiction
Multiplier
